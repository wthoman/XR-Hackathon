## Overview

SnapML allows you to incorporate machine learning models into your Spectacles applications. This workflow demonstrates how to adapt the Multi-Object Detection template from the [SnapML templates repository](https://github.com/Snapchat/snapml-templates/tree/main) using cloud-based development environments and specialized tools for dataset management and training.

The complete workflow consists of these major phases:

1. **Setup**: Prepare your development environment in Paperspace
2. **Data Preparation**: Manage and export your dataset with Roboflow
3. **Training**: Train a custom YOLOv7 model using your dataset
4. **Export**: Convert your trained model to ONNX format for SnapML
5. **Deployment**: Import your model into Lens Studio for use on Spectacles

### Quick Start Guide

For those familiar with machine learning workflows, here's a condensed version of the steps:

**Quick Start Workflow**

```bash
# 1. Set up environment in Paperspace with PyTorch template
git clone https://github.com/Snapchat/snapml-templates.git
git clone https://github.com/hartwoolery/yolov7
cd yolov7
git checkout export-snapml
pip install -r requirements.txt

# 2. Download dataset from Roboflow in YOLOv7 format
pip install --upgrade roboflow
# Use Roboflow API to download your dataset

# 3. Download pre-trained weights
wget https://github.com/WongKinYiu/yolov7/releases/download/v0.1/yolov7-tiny.pt

# 4. Train model
python train.py --data your-dataset/data.yaml --cfg cfg/training/yolov7-tiny.yaml \
  --weights yolov7-tiny.pt --img 224 224 --batch-size 64 --epochs 200 \
  --name detection --device 0 --hyp data/hyp.scratch.tiny.yaml

# 5. Export to ONNX for SnapML
python export.py --weights runs/train/detection/weights/best.pt --grid \
  --simplify --export-snapml --img-size 224 224 --max-wh 224

# 6. Import ONNX model to Lens Studio and set up class mappings
# See detailed steps below
```

This guide will walk through each of these steps in detail, explaining the concepts and providing troubleshooting tips along the way.

<CollapsibleStepBlock number={1} title="Setting Up Paperspace Environment">

2. **Select a template**
   - For YOLOv7 PyTorch notebooks, choose the "**PyTorch 1.12**" template
   - This template includes most dependencies you'll need for machine learning
3. **Choose hardware specifications**
   - Select a **GPU option** for faster training (Free CPU options are available but "first come, first serve")
   - For faster training, the P4000 or higher GPU options are recommended
4. **Clone the SnapML templates repository**

   ```bash
   git clone https://github.com/Snapchat/snapml-templates.git
   ```

5. **Navigate to the Multi-Object Detection folder**
   - Open the notebook file (.ipynb) in the cloned repository
   - The notebook contains all necessary code for the entire workflow

<details>
<summary><b>About Pre-trained Weights & Transfer Learning</b></summary>

Pre-trained weights are parameter values saved after a model has been trained on a large dataset like COCO (Common Objects in Context), which contains over 330,000 images with 80 object categories. Using these weights for transfer learning offers several advantages:

- **Faster convergence**: Training starts from an optimized state rather than random initialization
- **Better performance**: The model leverages patterns and features already learned from a larger dataset
- **Less data needed**: You can achieve good results with a smaller custom dataset
- **Reduced overfitting**: The model has already learned generalizable features

Transfer learning works by:

1. Starting with weights pre-trained on a large dataset
2. Replacing the final classification layer with your custom classes
3. Fine-tuning the entire network or just later layers with your data

For mobile devices like Spectacles, the smaller `yolov7-tiny` variant is recommended for real-time inference.

</details>

<CollapsibleStepBlock number={4} title="Training Your Model">

Now it's time to train your custom model using the pre-trained weights and your dataset. Model training is the most computationally intensive part of the process and can take from minutes to hours depending on your dataset size and hardware.

1. **(Optional) Set up TensorBoard for monitoring**

   TensorBoard provides real-time visualization of training metrics, allowing you to monitor progress and identify issues early.

   **TensorBoard Setup**

   ```python
   # In a notebook environment:
   # %reload_ext tensorboard
   # %tensorboard --logdir runs/train

   # In a terminal:
   tensorboard --logdir runs/train
   ```

2. **Configure and start training**

   Below is the core training command with key parameters explained:

   **Model Training Configuration**

   ```python
   import os
   os.environ['WANDB_MODE'] = 'disabled'  # Disable Weights & Biases logging

   # Key training parameters
   EPOCH      = 200        # Total number of training epochs
   BATCH_SIZE = 64         # Number of images processed at once
   IMG_SIZE   = 224        # Input image resolution (height, width)
   JOB_NAME   = 'detection'  # Name for this training run

   # Change directory to yolov7 folder
   # In terminal: cd ~/yolov7

   # Run the training command
   python train.py \
     --data    your-dataset-name-1/data.yaml \  # Path to dataset configuration
     --cfg     cfg/training/yolov7-tiny.yaml \  # Model architecture
     --weights yolov7-tiny.pt \                 # Pre-trained weights
     --img     224 224 \                        # Input resolution
     --batch-size 64 \                          # Batch size
     --epochs 200 \                             # Number of epochs
     --name    detection \                      # Run name
     --device  0 \                              # GPU device (0 = first GPU)
     --hyp     data/hyp.scratch.tiny.yaml       # Hyperparameters
   ```

    During training, you'll see lines like this in your console:

    ```
    Epoch   gpu_mem       box       obj       cls     total    labels  img_size
    0/199    0.367G   0.07126    0.3772         0    0.4485       188       224
    ```

    <details>
      <summary>
        <b>Training Metrics Explained</b>
      </summary>
      <div>
        <ul>
          <li>
            <strong>Epoch</strong>: Current training epoch / total epochs
          </li>
          <li>
            <strong>gpu_mem</strong>: GPU memory usage
          </li>
          <li>
            <strong>box</strong>: Bounding box regression loss (lower = better
            localization)
          </li>
          <li>
            <strong>obj</strong>: Objectness loss (detecting if there's any
            object)
          </li>
          <li>
            <strong>cls</strong>: Classification loss (which class the object
            belongs to)
          </li>
          <li>
            <strong>total</strong>: Sum of all losses (main training metric to
            reduce)
          </li>
          <li>
            <strong>labels</strong>: Number of ground truth objects in the batch
          </li>
          <li>
            <strong>img_size</strong>: Input image resolution
          </li>
        </ul>
        <p>
          Look for decreasing loss values as training progresses. Plateauing
          loss indicates the model may have converged.
        </p>
      </div>
    </details>

<CollapsibleStepBlock number={5} title="Exporting for SnapML">

To make the model compatible with Spectacles, you need to export it to the ONNX format that SnapML can use. ONNX (Open Neural Network Exchange) is an open standard for representing machine learning models, allowing for interoperability between different frameworks.

1.  **Install required packages**

    First, install the necessary dependencies for ONNX export:

    **ONNX Export Dependencies**

    ```python
    # Install dependencies for ONNX export
    pip install "protobuf<4.21.3"
    pip install "onnx>=1.9.0"
    pip install onnx-graphsurgeon
    pip install --user "onnx-simplifier>=0.3.6"
    ```

2.  **Export to ONNX format**

            Run the export script with the appropriate flags:

            **ONNX Export Command**

            ```python
            # Run export script for SnapML compatibility
            python export.py \
              --weights ./runs/train/detection/weights/best.pt \  # Path to trained weights
            ```

### Advanced Topics

#### Understanding Non-Maximum Suppression (NMS)

When exporting models for SnapML, you may notice the `--export-snapml` flag which affects how Non-Maximum Suppression (NMS) is handled:

<details>
  <summary><b>What is Non-Maximum Suppression?</b></summary>

NMS is not a trainable convolutional layer but a deterministic post-processing step applied to raw box predictions during inference. Its purpose is to:

1. **Collect all predicted boxes** whose confidence (objectness × class score) exceeds your threshold
2. **Sort** those boxes by confidence (highest to lowest)
3. **Keep only the best boxes** by removing highly overlapping detections of the same object

When exporting a YOLO model to ONNX for SnapML, we exclude NMS from the graph and re-implement it in Lens Studio using JavaScript. This approach provides more control over the detection process and ensures compatibility with Spectacles.

</details>

#### Model Optimization Tips

When creating models for Spectacles, consider these optimization strategies:

- **Use smaller variants** like YOLOv7-tiny for real-time performance
- **Reduce input image size** to balance detection quality and speed
- **Limit the number of classes** to improve model efficiency
- **Use quantization** when exporting if possible
- **Test thoroughly on-device** to ensure real-time performance

   ```typescript
     inputTexture: Texture; // Camera feed

     private processModelOutput(outputs: Record<string, MLTensor>): void {
       // YOLODetectionProcessor handles box decoding and NMS

       this.visualizer.updateDetections(detections);
     }
   }
   ```

<CollapsibleStepBlock number={3} title="Adding Smoothing for Better User Experience">

Raw ML detections can be jittery and unstable. Proper smoothing techniques create more natural AR experiences by stabilizing object placement.

1. **Motion Smoothing with MatchTransform**

   `MatchTransform` provides interpolation between positions, rotations, and scales:

   ```typescript
   // Example usage of MatchTransform
   @component
   export class MatchTransformTS extends BaseScriptComponent {
     @input
     target!: SceneObject; // Object to match

     @input
     usePositionLerp: boolean = true; // Enable smooth transitions

     @input
     positionLerpSpeed: number = 1.0; // Lower = smoother but slower

     // Implementation
     update() {
       if (this.usePositionLerp) {
         // Smooth position transition with lerp
         const currentPos = this.getSceneObject()
           .getTransform()
           .getLocalPosition();
         const targetPos = this.target.getTransform().getLocalPosition();

         // Calculate interpolated position
         const newPos = vec3.lerp(
           currentPos,
           targetPos,
           Math.min(1.0, this.positionLerpSpeed * getDeltaTime())
         );

         this.getSceneObject().getTransform().setLocalPosition(newPos);
       }
     }
   }
   ```

2. **Temporal Filtering in DetectionVisualizer**

   Implementing temporal filtering helps reduce jitter:

   ```typescript
   // Excerpt from temporal filtering implementation
   updateObjectPosition(index: number, newPosition: vec3, detection: Detection): void {
     // If we have a previous position, apply temporal smoothing
     if (this.previousPositions[index]) {
       // Calculate how much change is acceptable
       const moveThreshold = this.calculateMoveThreshold(detection);

       // If movement is small enough, apply smoothing
       if (this.getDistance(newPosition, this.previousPositions[index]) < moveThreshold) {
         // Apply heavier smoothing when movement is small
         newPosition = vec3.lerp(
           this.previousPositions[index],
           newPosition,
           0.3 // Lower value = more smoothing
         );
       } else {
         // Less smoothing for large movements
         newPosition = vec3.lerp(
           this.previousPositions[index],
           newPosition,
           0.7
         );
       }
     }

     // Update the position and store for next frame
     this.setObjectPosition(index, newPosition);
     this.previousPositions[index] = newPosition;
   }
   ```

3. **Position Stabilization with InBetween**

   `InBetween` calculates intermediate positions between objects:

   ```typescript
   // Using InBetween for stable positioning
   @component
   export class InBetween extends BaseScriptComponent {
     @input
     target1!: SceneObject; // First reference object

     @input
     target2!: SceneObject; // Second reference object

     @input
     positionBlend: number = 0.5; // Weight between targets (0.5 = halfway)

     update() {
       if (!this.target1 || !this.target2) return;

       // Get positions
       const pos1 = this.target1.getTransform().getWorldPosition();
       const pos2 = this.target2.getTransform().getWorldPosition();

       // Calculate weighted position
       const blendedPosition = vec3.lerp(pos1, pos2, this.positionBlend);

       // Apply position
       this.getSceneObject().getTransform().setWorldPosition(blendedPosition);
     }
   }
   ```

1. **Using SmartTether for Intelligent Object Placement**

   `SmartTether` keeps content appropriately positioned relative to the user:

   ```typescript
   // SmartTether implementation
   @component
   export class SmartTether extends BaseScriptComponent {
     @input
     verticalDistanceFromTarget: number = 0.1; // Vertical threshold

     @input
     horizontalDistanceFromTarget: number = 0.1; // Horizontal threshold

     update() {
       // Get current positions
       const targetPos = this.target.getTransform().getWorldPosition();
       const objectPos = this.getSceneObject()
         .getTransform()
         .getWorldPosition();

       // Check if object needs repositioning
       if (this.needsRepositioning(targetPos, objectPos)) {
         // Calculate new position based on target's view
   ```
